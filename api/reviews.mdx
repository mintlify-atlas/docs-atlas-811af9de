---
title: "Reviews"
description: "Submit and manage customer reviews"
---

Review endpoints allow customers to submit testimonials and admins to manage them. The list endpoint is public; write operations require authentication.

<Note>
  `POST`, `PUT`, and `DELETE` require a valid `Authorization: Bearer <token>` header.
</Note>

---

## GET /reviews

Returns a list of all customer reviews.

**Response**

<ResponseField name="status" type="string">
  `ok` on success.
</ResponseField>

<ResponseField name="data" type="array">
  Array of review objects. See [review object fields](#review-object) below.
</ResponseField>

**Example**

```bash
curl http://localhost:3042/reviews
```

**Success response**

```json
{
  "status": "ok",
  "data": [
    {
      "_id": "64f1a2b3c4d5e6f7a8b9c0d1",
      "author": "Lena P.",
      "content": "Best gym I have ever been to. The trainers are fantastic!",
      "position": "Member since 2022",
      "createdAt": "2024-01-15T10:00:00.000Z",
      "updatedAt": "2024-01-15T10:00:00.000Z"
    }
  ]
}
```

---

## GET /review/:id

Fetch a single review by its MongoDB ObjectId.

<ParamField path="id" type="string" required>
  MongoDB ObjectId of the review.
</ParamField>

**Example**

```bash
curl http://localhost:3042/review/64f1a2b3c4d5e6f7a8b9c0d1
```

---

## POST /review

Submit a new customer review.

<ParamField body="author" type="string" required>
  Name of the reviewer.
</ParamField>

<ParamField body="content" type="string" required>
  The review text.
</ParamField>

<ParamField body="position" type="string">
  Optional subtitle shown under the author's name, e.g. `Member since 2022`.
</ParamField>

**Response** — `201 Created`

<ResponseField name="status" type="string">
  `ok` on success.
</ResponseField>

<ResponseField name="message" type="string">
  `Review created successfully`
</ResponseField>

<ResponseField name="data" type="object">
  The newly created review object.
</ResponseField>

**Example**

```bash
curl -X POST http://localhost:3042/review \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "author": "Thomas K.",
    "content": "The morning HIIT classes are incredible. Highly recommended!",
    "position": "Member since 2023"
  }'
```

---

## PUT /review

Update an existing review. At least one of `author` or `content` must be provided.

<ParamField body="id" type="string" required>
  MongoDB ObjectId of the review to update.
</ParamField>

<ParamField body="author" type="string">
  Updated author name.
</ParamField>

<ParamField body="content" type="string">
  Updated review text.
</ParamField>

<ParamField body="position" type="string">
  Updated position/subtitle.
</ParamField>

**Example**

```bash
curl -X PUT http://localhost:3042/review \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"id": "64f1a2b3c4d5e6f7a8b9c0d1", "content": "Still the best gym in town after two years!"}'
```

---

## DELETE /review/:id

Permanently delete a review.

<ParamField path="id" type="string" required>
  MongoDB ObjectId of the review to delete.
</ParamField>

**Example**

```bash
curl -X DELETE http://localhost:3042/review/64f1a2b3c4d5e6f7a8b9c0d1 \
  -H "Authorization: Bearer <token>"
```

**Success response**

```json
{
  "status": "ok",
  "message": "Review deleted successfully",
  "data": { "_id": "64f1a2b3c4d5e6f7a8b9c0d1", "author": "Thomas K." }
}
```

## Review object

<ResponseField name="_id" type="string">
  MongoDB ObjectId.
</ResponseField>

<ResponseField name="author" type="string">
  Name of the reviewer.
</ResponseField>

<ResponseField name="content" type="string">
  Review text.
</ResponseField>

<ResponseField name="position" type="string">
  Optional subtitle, e.g. job title or membership tenure.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 creation timestamp.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 last-updated timestamp.
</ResponseField>
