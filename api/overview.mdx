The Xtreme Fitness API is a RESTful JSON API that powers the backend of the platform. All requests and responses use `application/json`.

## Base URL

```
http://localhost:3042
```

## Authentication

Most write operations require a valid JWT. The token can be supplied in one of two ways:

**Authorization header (preferred)**

```
Authorization: Bearer <token>
```

**Request body**

Some endpoints (e.g. `POST /auth/token`) also accept `{ "token": "<jwt>" }` in the JSON body.

Obtain a token by calling [`POST /auth/signin`](/api/auth#post-authsignin).

<Info>
  The `USE_JWT` environment variable controls whether the auth middleware is enforced. When set to `false`, protected routes are accessible without a token. This should never be disabled in production.
</Info>

## Content-Type

All request bodies must use `Content-Type: application/json` unless the endpoint accepts file uploads, in which case use `multipart/form-data`.

## Standard response envelope

Every response is wrapped in a consistent envelope:

```json
{
  "status": "ok" | "error" | "not_found",
  "message": "Human-readable description",
  "data": {} | []
}
```

| Field | Type | Description |
|---|---|---|
| `status` | `string` | `ok` on success, `error` on failure, `not_found` when the resource does not exist |
| `message` | `string` | Human-readable description of the result |
| `data` | `object` or `array` | The returned resource(s), or an empty array on error |

## Error responses

```json
{
  "status": "error",
  "message": "No access without token."
}
```

Common HTTP status codes:

| Status | Meaning |
|---|---|
| `200` | Success |
| `201` | Resource created |
| `400` | Validation error — missing or invalid fields |
| `401` | Token invalid or expired |
| `404` | Resource not found |
| `500` | Internal server error |
