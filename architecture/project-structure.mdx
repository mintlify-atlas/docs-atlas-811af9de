### `src/components/`

```text
components/
в”њв”Ђв”Ђ Layout.jsx              # Shared page shell: renders Navbar and <Outlet> for public routes
в”њв”Ђв”Ђ Navbar.jsx              # Site-wide navigation bar
в””в”Ђв”Ђ ui/                     # Low-level Radix + Tailwind component primitives (buttons, inputs, etc.)
```

`Layout.jsx` is the route element that wraps all public marketing pages. It renders the `Navbar` at the top and an `<Outlet>` for the matched child route below it.

### `src/lib/`

```text
lib/
в”њв”Ђв”Ђ api/
в”‚   в”њв”Ђв”Ђ client.js           # Fetch wrapper, token storage helpers, Authorization header injection
в”‚   в”њв”Ђв”Ђ hooks.js            # TanStack Query hooks for all resources

в”‚   в””в”Ђв”Ђ auth-context.jsx    # AuthProvider, useAuth hook, JWT decode/validate, session restore
в”њв”Ђв”Ђ content/                # Static or CMS-sourced content helpers
в”њв”Ђв”Ђ rich-text/              # Plate editor configuration and serialiser utilities
```

`auth-context.jsx` is the single source of truth for authentication state. It exposes `signIn`, `signOut`, `refreshSession`, `isAuthenticated`, `isAdmin`, and `isMember` to the rest of the app.

### `src/features/`

Features are self-contained slices of functionality that are too complex to live in `screens/` or `components/` alone.

```text
features/
в”њв”Ђв”Ђ admin/
в”‚   в”њв”Ђв”Ђ components/
в”‚   в”‚   в”њв”Ђв”Ђ AdminLayout.jsx     # Admin shell: sidebar navigation + <Outlet>
в”‚   в”‚   в””в”Ђв”Ђ RequireAuth.jsx     # Route guard вЂ” checks authentication and role before rendering
в”‚   в”њв”Ђв”Ђ forms/                  # Admin-specific form components (blog editor, etc.)
в”‚   в”њв”Ђв”Ђ screens/
в”‚   в”‚   в”њв”Ђв”Ђ AdminDashboard      # /admin вЂ” Overview and quick stats
в”‚   в”‚   в”њв”Ђв”Ђ AdminBlogs          # /admin/blogs вЂ” Create, edit, and delete blog posts
в”‚   в”‚   в”њв”Ђв”Ђ AdminMessages       # /admin/messages вЂ” View contact form submissions
в”‚   в”‚   в””в”Ђв”Ђ AdminReviews        # /admin/reviews вЂ” Moderate member reviews
в”‚   в””в”Ђв”Ђ ui.jsx                  # Admin-scoped UI component re-exports
в””в”Ђв”Ђ marketing/
    в”њв”Ђв”Ђ content/                # Static content definitions for marketing sections
    в”њв”Ђв”Ђ sections/               # Reusable marketing section components (hero, CTA, etc.)
```

`RequireAuth` reads `isAuthenticated` and `user.role` from `useAuth()`. Unauthenticated visitors are redirected to `/login`. Authenticated users whose role does not match `allowRoles` are redirected to the area appropriate for their role.

### `backend/lib/handlers/`

Handlers contain the actual business logic for each resource. Routes import handlers and pass them as Express route callbacks. This separation keeps route files thin and makes the logic easier to test in isolation.

<Tip>
The `user` model enforces that `admin` and `guest` accounts cannot hold `subscription` or `enrolledWorkouts` values. This is enforced at the Mongoose level via `pre('save')`, `pre('findOneAndUpdate')`, and schema-level validators вЂ” not just in application code.
</Tip>
