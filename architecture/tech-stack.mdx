## Frontend

<CardGroup cols={2}>
  <Card title="React 19" icon="react">
    The UI layer. React 19 is used for its concurrent rendering model and the `use` API. All screens are lazy-loaded via `React.lazy()` to keep the initial bundle small.
  </Card>
  <Card title="Vite 7" icon="bolt">
    The build tool and dev server. Vite's native ES module approach means near-instant hot-module replacement during development and fast production builds via Rollup. The React plugin (`@vitejs/plugin-react`) handles JSX and Fast Refresh.
  </Card>
  <Card title="React Router v7" icon="route">
    Client-side routing. React Router v7 provides the `BrowserRouter`, `Routes`, `Route`, `Navigate`, and `Outlet` primitives used throughout the app. The single route declaration in `src/index.jsx` defines all public, admin, and member routes.
  </Card>
  <Card title="TanStack Query v5" icon="database">
    Server-state management. TanStack Query handles fetching, caching, background refetching, and error states for all API calls. A single `QueryClient` instance is created at the root and shared across public and admin routes.
  </Card>
  <Card title="Tailwind CSS v4" icon="palette">
    Utility-first styling. Tailwind v4 is integrated through its Vite plugin (`@tailwindcss/vite`), which processes CSS at build time. The `tw-animate-css` package adds animation utilities and `tailwind-merge` resolves conflicting class names at runtime.
  </Card>
  <Card title="React Hook Form v7 + Zod v4" icon="clipboard-check">
    Form management and validation. React Hook Form minimises re-renders by keeping form state uncontrolled. Zod schemas describe the expected shape and constraints of form data; `@hookform/resolvers` connects the two libraries so validation runs on submit.
  </Card>
  <Card title="Plate v52" icon="file-text">
    Rich-text editing. The Plate editor framework (`platejs`, `@platejs/basic-nodes`, `@platejs/link`, `@platejs/list`) powers the blog post editor in the admin section. `dompurify` sanitises HTML before it is rendered from stored rich-text content.
  </Card>
  <Card title="Radix UI" icon="layout">
    Headless, accessible component primitives. `@radix-ui/react-navigation-menu`, `@radix-ui/react-separator`, and `@radix-ui/react-slot` provide unstyled interactive components that are then styled with Tailwind. `class-variance-authority` and `clsx` manage conditional class composition on top of these primitives.
  </Card>
</CardGroup>

## Backend

<CardGroup cols={2}>
  <Card title="Express 5" icon="server">
    The HTTP framework. Express 5 handles routing, middleware composition, and static file serving. The server is configured in `backend/lib/server.js` and exported as a module so it can be imported by the entry point `backend/index.js`.
  </Card>
  <Card title="MongoDB + Mongoose 8" icon="database">
    The database and ODM. MongoDB stores all application data. Mongoose provides schema definitions, validation hooks, and a consistent query interface. Models live in `backend/lib/db/models/`.
  </Card>
  <Card title="jsonwebtoken 9" icon="key">
    JWT creation and verification. `jwt.sign()` issues tokens on successful login; `jwt.verify()` validates them inside the auth middleware (`backend/lib/middleware/auth.middleware.js`) before protected routes are reached.
  </Card>
  <Card title="bcrypt / bcryptjs" icon="lock">
    Password hashing. Passwords are never stored in plain text. `bcryptjs` is used to compare a submitted password against the stored hash during sign-in.
  </Card>
  <Card title="multer 2" icon="upload">
    File upload handling. `multer` processes `multipart/form-data` requests, making uploaded files available to route handlers. Used for profile pictures and any other binary assets.
  </Card>
  <Card title="cors, cookie-parser, body-parser" icon="shield">
    Request processing middleware. `cors` allows the frontend dev server (a different origin) to call the API during development. `cookie-parser` makes cookies available on `req.cookies`. `body-parser` parses JSON and URL-encoded request bodies.
  </Card>
</CardGroup>

## Dev tooling

**Vite** serves as both the development server and production bundler for the frontend. It provides fast cold starts, instant HMR, and optimised output with code-splitting by route.

**nodemon** watches the backend source for changes and automatically restarts the Node process during development, eliminating the need to manually restart the server after every edit.

**ESLint** is configured in both the frontend and backend workspaces. The backend uses `@eslint/js` with flat config (`backend/.eslintrc.json`). ESLint keeps code consistent and catches common mistakes before they reach the repository.
